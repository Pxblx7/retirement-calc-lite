# Cmm Lite

## Instructions

1. Add clear inline comments for every code block you modify, explaining both the 'Why' (rationale) and the 'What' (logic).
2. Automatically update the `README.md` after every significant change, documenting what was changed, why it was changed, and which files were affected.
3. Proactively suggest structural improvements or refactoring if you detect technical debt or inefficient patterns.
4. Maintain strict consistency with the existing project's formatting, naming conventions, and TypeScript types.
5. Do not alter the core project architecture (e.g., moving folders or changing build tools) without explicit authorization.
6. Always ask for confirmation and provide a rationale before adding any new npm dependencies or external packages.
